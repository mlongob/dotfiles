# Global Claude Code Instructions

This file is loaded by Claude Code in every project.

## General preferences

- Be concise. Lead with the answer or action, skip preamble.
- No trailing summaries after completing a task — the diff speaks for itself.
- No emojis unless explicitly requested.
- Prefer editing existing files over creating new ones.
- Don't add comments, docstrings, or type annotations to code that wasn't changed.
